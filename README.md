## Why This Documentation?

This repository contains the **official Pyrodactyl documentation** site. It is designed to help users, administrators, and contributors find accurate information quickly and keep the project easy to understand.

- **Clear and organized**: Content is structured into logical sections so you can find setup guides, references, and explanations without digging through unrelated pages.

- **Fast to browse**: The documentation is built for quick navigation, lightweight page loads, and a smooth reading experience.

- **Accessible for everyone**: Keyboard navigation, screen reader support, semantic markup, and readable layouts make the docs usable for all readers.

- **Easy to maintain**: Markdown and MDX-based content make it simple to update pages, add examples, and keep information current.

- **Contributor-friendly**: The structure is designed so new contributors can add or improve documentation without needing to understand a complex workflow.

- **Responsive everywhere**: Read the documentation comfortably on desktop, tablet, or mobile devices.

- **Built on modern tooling**: Next.js, React, and TypeScript provide a reliable foundation for a maintainable documentation experience.

- **Ready to grow**: The documentation is built to expand alongside Pyrodactyl as new features, workflows, and guides are added.

## Core Features

This documentation site is built to make information easy to discover, read, and maintain:

* Structured guides and reference pages

* Built-in search for quick lookup

* MDX support for rich content and examples

* Consistent navigation across all sections

* Code blocks, callouts, and formatted examples

* Content that is easy to extend and update

* A layout designed for both beginners and advanced users
