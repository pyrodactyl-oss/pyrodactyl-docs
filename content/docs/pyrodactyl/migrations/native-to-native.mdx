## Migrating from Pterodactyl (Native → Native)

<Callout type='error'>
Once you switch to Pyrodactyl, switching back to Pterodactyl is not supported and may cause errors as Pyrodactyl modifies the Pterodactyl Database Schema. Provided below is how you can make a backup of your Panel Database if you need to switch back. It is also recommended that you make a backup of the Pterodactyl folder aswell if anything goes wrong during migration. (Replace ``panel`` with the database name you use)

```bash
# If using MySQL
mysqldump -u root -p --opt panel > ./panel.sql

# If using MariaDB
mariadb-dump -u root -p --opt panel > ./panel.sql
```
</Callout>

### Prerequisites

- Root or sudo access on your Linux server
- Latest [NodeJS LTS](https://nodejs.org/en) installed
- Composer installed globally
- MySQL database access and credentials
- Backup of `.env` and your current database

### Migration Steps

<Steps>
<Step>
### Install Node Version Manager

```bash
curl -o- https://fnm.vercel.app/install | bash
```

</Step>

<Step>
### Install Node 22

```bash
fnm install 22
```

</Step>

<Step>
### Navigate to Pterodactyl Directory

```bash
cd /var/www/pterodactyl
```

</Step>

<Step>
### Enable Maintenance Mode

```bash
php artisan down
```

</Step>

<Step>
### Download Pyrodactyl Release

```bash
curl -L https://github.com/pyrodactyl-oss/pyrodactyl/releases/latest/download/panel.tar.gz | tar -xzv
```

</Step>

<Step>
### Set Permissions

```bash
chmod -R 755 storage/* bootstrap/cache
```

</Step>

<Step>
### Install PHP Dependencies

```bash
composer install --no-dev --optimize-autoloader
```

</Step>

<Step>
### Clear Cached Views and Config

```bash
php artisan view:clear
php artisan config:clear
```

</Step>

<Step>
### Run Database Migrations

```bash
php artisan migrate --seed --force
```

</Step>

<Step>
### Set Ownership

```bash
chown -R www-data:www-data /var/www/pterodactyl/*
```

</Step>

<Step>
### Restart Queue Worker

```bash
php artisan queue:restart
```

</Step>

<Step>
### Build Frontend Assets

```bash
npm install -g corepack@latest
corepack enable pnpm
pnpm i --frozen-lockfile && pnpm run ship
chmod -R 755 storage/* bootstrap/cache/
```

</Step>

<Step>
### Bring Panel Back Online

```bash
php artisan up
```

</Step>
</Steps>

🎉 **Boom! Your panel is now migrated to Pyrodactyl!**
