## Updating Your Native Installation

Newest release version: 

![Latest Release](https://shieldcn.dev/github/release/pyrodactyl-oss/pyrodactyl.svg?variant=branded)

<Callout type="warn">
  Before proceeding with any updates, make sure to back up your database and panel files. Provided below is how you can make a backup of your Panel Database. (Replace ``panel`` with the database name you use)

```bash
# If using MySQL
mysqldump -u root -p --opt panel > ./panel.sql

# If using MariaDB
mariadb-dump -u root -p --opt panel > ./panel.sql
```
</Callout>

### Prerequisites

- Root or sudo access on your server
- Latest [NodeJS LTS](https://nodejs.org/en) installed
- Composer installed globally
- Backup of your current installation

### Update Steps

<Steps>
<Step>
### Install Node Version Manager

```bash
curl -o- https://fnm.vercel.app/install | bash
```

</Step>

<Step>
### Install Node 22

```bash
fnm install 22
```

</Step>

<Step>
### Navigate to Pyrodactyl Directory

```bash
cd /path/to/your/pyrodactyl
```

</Step>

<Step>
### Download Latest Release

```bash
curl -L https://github.com/pyrodactyl-oss/pyrodactyl/releases/latest/download/panel.tar.gz | tar -xzv
```

</Step>

<Step>
### Enable Maintenance Mode

```bash
php artisan down
```

</Step>

<Step>
### Set Proper Permissions

```bash
chmod -R 755 storage/* bootstrap/cache
```

</Step>

<Step>
### Install PHP Dependencies

```bash
composer install --no-dev --optimize-autoloader
```

</Step>

<Step>
### Clear Application Cache

```bash
php artisan view:clear
php artisan config:clear
```

</Step>

<Step>
### Run Database Migrations

```bash
php artisan migrate --seed --force
```

</Step>

<Step>
### Set Ownership

```bash
# For NGINX (non-CentOS)
chown -R www-data:www-data /var/www/pterodactyl/*

# For NGINX (CentOS)
chown -R nginx:nginx /var/www/pterodactyl/*

# For Apache (CentOS)
chown -R apache:apache /var/www/pterodactyl/*
```

</Step>

<Step>
### Restart Queue Worker

```bash
php artisan queue:restart
```

</Step>

<Step>
### Build Frontend Assets

```bash
npm install -g corepack@latest
corepack enable pnpm
pnpm i --frozen-lockfile && pnpm run ship
```

</Step>

<Step>
### Verify Permissions

```bash
chmod -R 755 storage/* bootstrap/cache/
```

</Step>

<Step>
### Disable Maintenance Mode

```bash
php artisan up
```

</Step>
</Steps>

### Troubleshooting Common Issues

#### Hydration Mismatch Errors

If you encounter hydration mismatch errors in the browser console, try these steps:

<Steps>
<Step>
### Clear Browser Cache

Clear your browser cache and cookies.
</Step>

<Step>
### Rebuild Frontend Assets

```bash
pnpm i --frozen-lockfile && pnpm run ship
```

</Step>

<Step>
### Clear Application Cache

```bash
php artisan view:clear
php artisan config:clear
```

</Step>
</Steps>

<Callout type="info">
  Hydration mismatches often occur when the server-rendered content doesn't
  match the client-side rendered content. This can happen due to browser
  extensions or cached assets.
</Callout>

#### Permission Issues

If you encounter permission issues after updating:

1. Verify the ownership of your files matches your web server user
1. Ensure storage and cache directories are writable:

   ```bash
   chmod -R 755 storage/* bootstrap/cache/
   chown -R www-data:www-data storage bootstrap/cache
   ```

<Callout type="warn">
  Replace `www-data` with your web server user (nginx, apache) if different.
</Callout>
