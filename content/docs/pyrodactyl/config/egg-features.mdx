---
title: Egg Features
description: Feature flags supported by Pyrodactyl eggs.
---

Egg features enable panel behavior for servers that support it. Add the feature
string to the egg so Pyrodactyl can expose the matching UI.

## General features

`hytale_oauth`: Shows the Hytale setup modal for downloading and configuring the
Hytale server binary.

`mclogs`: Adds mclo.gs crash and log analysis for Minecraft servers.

## Subdomains

Use a subdomain feature to allow Pyrodactyl to create server connection records
for the egg.

- `subdomain_minecraft`: Minecraft servers
- `subdomain_factorio`: Factorio servers
- `subdomain_rust`: Rust servers
- `subdomain_scpsl`: SCP: Secret Laboratory servers
- `subdomain_teamspeak`: TeamSpeak servers

See [Cloudflare Subdomains](/docs/pyrodactyl/subdomains/cloudflare) for DNS
provider setup.

## Minecraft mods and plugins

Mod manager features use a `mod/` or `plugin/` prefix:

- `mod/*` installs to `mods/`
- `plugin/*` installs to `plugins/`

The value after the prefix is matched against Modrinth loaders. For example,
`plugin/velocity` makes the mod manager search for Velocity-compatible plugins.

Datapacks do not need a feature flag. Pyrodactyl shows datapacks for
vanilla-derived Minecraft servers and installs them to `world/datapacks/`.

### Mod loaders

- `mod/fabric`: Fabric mods
- `mod/forge`: Forge mods
- `mod/neoforge`: NeoForge mods
- `mod/quilt`: Quilt mods

### Plugin platforms

- `plugin/paper`: Paper plugins
- `plugin/spigot`: Spigot plugins
- `plugin/bukkit`: Bukkit plugins
- `plugin/purpur`: Purpur plugins
- `plugin/velocity`: Velocity proxy plugins
- `plugin/waterfall`: Waterfall proxy plugins
- `plugin/bungeecord`: BungeeCord proxy plugins

<Callout type="info">
  Some platforms can run plugins from related loaders. For example, Paper can
  run many Spigot and Bukkit plugins, so Pyrodactyl includes those results for
  Paper servers.
</Callout>

See [Mod Manager](/docs/pyrodactyl/mod-manager) for user-facing mod manager
behavior.
