# Troubleshooting

This guide covers common issues users encounter with Pyrodactyl and their solutions.

## Installation Issues

### Docker Container Won't Start

**Problem**: Docker container exits immediately or fails to start.

**Solutions**:
1. Check container logs:
   ```bash
   docker logs pyrodactyl-panel
   docker logs pyrodactyl-wings
   ```

2. Verify port conflicts:
   ```bash
   netstat -tulpn | grep :8080
   netstat -tulpn | grep :443
   ```

3. Check docker-compose.yml configuration:
   - Ensure volume paths exist
   - Verify environment variables are correctly set
   - Check network configuration

### Database Connection Failed

**Problem**: Panel cannot connect to the database.

**Solutions**:
1. Verify database is running:
   ```bash
   docker ps | grep database
   ```

2. Check database credentials in `.env`:
   ```bash
   DB_CONNECTION=mysql
   DB_HOST=database
   DB_PORT=3306
   DB_DATABASE=pyrodactyl
   DB_USERNAME=pyrodactyl
   DB_PASSWORD=your-password
   ```

3. Test database connection:
   ```bash
   docker exec -it pyrodactyl-panel php artisan tinker
   >>> DB::connection()->getPdo()
   ```

### Permission Errors

**Problem**: File permission errors during installation.

**Solutions**:
1. Set correct ownership:
   ```bash
   sudo chown -R www-data:www-data /var/www/pyrodactyl
   sudo chmod -R 755 /var/www/pyrodactyl/storage
   sudo chmod -R 755 /var/www/pyrodactyl/bootstrap/cache
   ```

2. Check SELinux status:
   ```bash
   sudo setenforce 0
   ```

## Panel Issues

### 502 Bad Gateway

**Problem**: Panel returns 502 error.

**Solutions**:
1. Check PHP-FPM status:
   ```bash
   docker exec -it pyrodactyl-panel supervisorctl status php-fpm
   ```

2. Restart services:
   ```bash
   docker-compose restart panel
   ```

3. Check nginx configuration:
   ```bash
   docker exec -it pyrodactyl-panel nginx -t
   ```

### Login Loop

**Problem**: Users get stuck in login loop after authentication.

**Solutions**:
1. Clear session cache:
   ```bash
   docker exec -it pyrodactyl-panel php artisan cache:clear
   docker exec -it pyrodactyl-panel php artisan config:clear
   ```

2. Check session configuration:
   ```bash
   SESSION_DRIVER=database
   SESSION_SECURE_COOKIES=true
   ```

3. Verify SSL certificate is valid.

### Server Not Showing in Panel

**Problem**: Created servers don't appear in the panel.

**Solutions**:
1. Check Elytra connection:
   ```bash
   docker exec -it pyrodactyl-elytra elytra debug
   ```

2. Verify daemon configuration:
   - Check API token matches
   - Ensure communication URL is accessible
   - Verify SSL certificate

3. Restart Elytra daemon:
   ```bash
   docker-compose restart elytra
   ```

## Server Issues

### Server Won't Start

**Problem**: Server creation succeeds but server won't start.

**Solutions**:
1. Check server logs in the panel
2. Verify server configuration:
   - Memory allocation
   - CPU allocation
   - Disk space
   - Port allocation

3. Check Docker container status:
   ```bash
   docker exec -it pyrodactyl-elytra docker ps -a
   ```

4. Inspect server container logs:
   ```bash
   docker logs [container-id]
   ```

### Installation Stuck

**Problem**: Server installation process gets stuck.

**Solutions**:
1. Check available disk space on the node
2. Verify download URLs in egg configuration
3. Check network connectivity
4. Restart installation:
   ```bash
   # From panel
   Settings → Server → Reinstall Server
   ```

### Console Not Working

**Problem**: Server console doesn't update or respond to commands.

**Solutions**:
1. Check WebSocket connection:
   - Open browser developer tools
   - Check Network tab for WebSocket errors

2. Verify Elytra daemon is running
3. Check firewall settings:
   ```bash
   sudo ufw status
   sudo ufw allow 8080
   ```

## Backup Issues

### Backup Failed

**Problem**: Server backups fail to complete.

**Solutions**:
1. Check backup configuration:
   ```bash
   BACKUP_DRIVER=s3  # or local
   ```

2. Verify storage credentials
3. Check available disk space
4. Review backup logs:
   ```bash
   docker exec -it pyrodactyl-elytra tail -f /var/log/elytra/backup.log
   ```

### Large Backup Times

**Problem**: Backups take extremely long to complete.

**Solutions**:
1. Use `.pyroignore` file to exclude large directories
2. Configure backup compression:
   ```bash
   BACKUP_COMPRESSION_LEVEL=6
   ```
3. Increase backup timeout:
   ```bash
   BACKUP_TIMEOUT=3600
   ```

## Performance Issues

### Slow Panel Loading

**Problem**: Panel interface loads slowly.

**Solutions**:
1. Enable caching:
   ```bash
   CACHE_DRIVER=redis
   ```

2. Optimize database:
   ```bash
   docker exec -it pyrodactyl-panel php artisan optimize:clear
   docker exec -it pyrodactyl-panel php artisan optimize
   ```

3. Check resource usage:
   ```bash
   docker stats pyrodactyl-panel
   ```

### High Memory Usage

**Problem**: Panel or Wings using excessive memory.

**Solutions**:
1. Monitor resource usage:
   ```bash
   docker stats --no-stream
   ```

2. Adjust PHP memory limit:
   ```bash
   PHP_MEMORY_LIMIT=256M
   ```

3. Limit server allocations appropriately

## Network Issues

### Node Communication Failed

**Problem**: Panel cannot communicate with Wings nodes.

**Solutions**:
1. Test connectivity:
   ```bash
   curl -k https://node-domain:8080/api/system
   ```

2. Check firewall rules:
   ```bash
   sudo ufw allow 8080/tcp
   ```

3. Verify SSL certificates
4. Check DNS resolution

### Port Allocation Issues

**Problem**: Unable to assign ports to servers.

**Solutions**:
1. Check available ports:
   ```bash
   # From Wings node
   netstat -tulpn | grep :25565
   ```

2. Add more allocations:
   - Admin → Nodes → Select Node → Allocations
   - Add new port ranges

3. Check port conflicts with other services

## Security Issues

### SSL Certificate Errors

**Problem**: SSL/HTTPS certificate issues.

**Solutions**:
1. Check certificate validity:
   ```bash
   openssl x509 -in /path/to/cert.pem -text -noout
   ```

2. Renew Let's Encrypt certificate:
   ```bash
   docker exec -it pyrodactyl-panel certbot renew
   ```

3. Use self-signed certificates for development only

### Authentication Issues

**Problem**: Users cannot authenticate properly.

**Solutions**:
1. Check 2FA configuration
2. Reset user passwords:
   ```bash
   docker exec -it pyrodactyl-panel php artisan tinker
   >>> $user = App\Models\User::find(1);
   >>> $user->password = Hash::make('new-password');
   >>> $user->save();
   ```

3. Verify email configuration for password resets

## Debug Mode

### Enabling Debug Mode

**Warning**: Only enable debug mode in development environments.

```bash
APP_DEBUG=true
APP_LOG_LEVEL=debug
```

### Debugging Steps

1. Check application logs:
   ```bash
   docker exec -it pyrodactyl-panel tail -f storage/logs/laravel.log
   ```

2. Monitor Elytra logs:
   ```bash
   docker exec -it pyrodactyl-elytra tail -f /var/log/elytra/elytra.log
   ```

3. Use browser developer tools:
   - Network tab for API calls
   - Console tab for JavaScript errors
   - Application tab for storage issues
